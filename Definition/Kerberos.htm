In Active Directory, a Golden Ticket is a type of forged Kerberos Ticket Granting Ticket (TGT) that is created using the KRBTGT account's password hash. The KRBTGT account is a service account that is used by the Key Distribution Centre (KDC) to encrypt TGTs for other accounts.

 

A Golden Ticket is a powerful tool that allows an attacker to authenticate as any user or service within an Active Directory domain, without actually needing to know the user's or service's password. This is because the attacker has used the KRBTGT account's password hash to create a forged TGT that grants them access to the Kerberos realm as any user or service they choose.

 

An attacker with a Golden Ticket can impersonate any domain user or service, access all resources that are protected by Kerberos, and maintain persistent access to the domain, even if the user or service's password is changed. Therefore, preventing the creation and use of Golden Tickets is critical for maintaining the security of an Active Directory environment.

 

To prevent Golden Ticket attacks, it is important to protect the KRBTGT account's password hash and implement measures such as regular password changes, secure storage of sensitive domain data, and effective monitoring and detection of suspicious activity within the environment.

Sliver Ticket: No detection because there is no interaction with the DC.

 

A Silver Ticket attack is a type of cyber-attack that involves forging a Kerberos Ticket Granting Ticket (TGT) for a specific service in an Active Directory environment. Unlike a Golden Ticket attack, which uses the KRBTGT account's password hash to create a forged TGT, a Silver Ticket attack leverages service account credentials to create the forged TGT.

 

An attacker who successfully executes a Silver Ticket attack can use the forged TGT to impersonate the targeted service, giving them access to the service's resources and potentially allowing them to move laterally throughout the environment. This attack can be particularly dangerous if the targeted service is highly privileged, such as a domain administrator account.

Skeleton Key Attack: A Skeleton Key Attack is a type of cyber-attack that involves exploiting a vulnerability in Microsoft's Active Directory (AD) authentication system. This attack involves the use of a backdoor password that is added to an AD domain's credentials cache, allowing the attacker to bypass normal authentication procedures and access any account in the domain without knowing the account's password. The Skeleton Key Attack is performed by gaining administrative access to an AD domain controller and injecting the backdoor password into the domain's credentials cache. Once the backdoor password has been injected, the attacker can use it to authenticate to any account in the domain by simply appending the password to the account's username.

 

This attack can be particularly dangerous because it allows the attacker to gain unauthorized access to any account in the domain, including highly privileged accounts such as domain administrators. Additionally, the attacker can maintain access to the domain even if the domain's administrator passwords are changed.

DSRM:- DSRM stands for Directory Services Restore Mode (safe mode), which is a mode in Microsoft's Active Directory Domain Services (AD DS) that allows administrators to repair or restore an AD DS domain controller. A DSRM attack is a type of cyber-attack that targets domain controllers in DSRM mode to gain unauthorized access to an AD DS environment.

 

In a DSRM attack, an attacker gains access to a domain controller in DSRM mode and then uses administrative privileges to elevate their access within the domain. Since domain controllers are critical components of an AD DS environment, compromising a domain controller through a DSRM attack can lead to significant damage to an organization's data and systems.
