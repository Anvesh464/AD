<p class=MsoNormal><b>Active Directory: </b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l2 level1 lfo1;tab-stops:list .5in'>Active
     directory is a centralized device which we used for managing the
     organization devices and it is centralized and secure management of an
     entire network.<o:p></o:p></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo1;tab-stops:list .5in'>It
     stores information about objects on the network and makes it easily available
     to users and admins<o:p></o:p></li>
 <li class=MsoNormal style='mso-list:l2 level1 lfo1;tab-stops:list .5in'>Directory
     service used to <span class=SpellE><span class=GramE>managed</span></span>
     the windows network.<o:p></o:p></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>Objective like <o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>AD --&gt; windows server --&gt; Account information,
privileges, profiles policy.<o:p></o:p></p>

<p class=MsoNormal>Windows Servers --&gt; management profiles, network
information, printers, file shares policy.<o:p></o:p></p>

<p class=MsoNormal>Network Devices: Configuration, quality of service policy,
security policy.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><b>Components: </b><span lang=EN-IN style='mso-ansi-language:
EN-IN'>Active Directory (AD) is composed of several different components, each
of which plays a specific role in the management and operation of the directory
service. The main components of AD include:</span><o:p></o:p></p>

<p class=MsoNormal><span lang=EN-IN style='mso-ansi-language:EN-IN'>&nbsp;<o:p></o:p></span></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l1 level1 lfo2;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Domain</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: A domain is a logical group of
     objects, such as users, computers, and printers, that share a common
     directory database. A domain is identified by a unique name and is managed
     as a single entity with its own policies and security settings.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l4 level1 lfo3;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Domain Controller</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: A domain controller is a
     server that manages user authentication and authorization, and stores
     information about objects in the domain. Each domain has at least one
     domain controller, which is responsible for managing and replicating
     directory information to other domain controllers in the same domain.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l5 level1 lfo4;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Active Directory Schema</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: The AD schema defines the
     structure and attributes of objects in the directory. The schema is stored
     on each domain controller and is replicated to all other domain
     controllers in the domain.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l0 level1 lfo5;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Forest</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: A forest is a collection of
     one or more domains that share a common schema and global <span
     class=SpellE>Catalog</span>. A forest is a security boundary and provides
     a mechanism for defining trust relationships between domains.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l9 level1 lfo6;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Global <span class=SpellE>Catalog</span></span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: The global <span
     class=SpellE>catalog</span> is a distributed data store that contains a
     subset of directory information from all domains in the forest. It
     provides a mechanism for searching and locating objects across domains.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l8 level1 lfo7;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Organizational Unit (OU</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>): An organizational unit is a
     container object that can be used to group and organize other objects in
     the directory, such as users, computers, and groups. OUs provide a
     mechanism for delegating administrative control and applying policies to a
     specific subset of objects.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l3 level1 lfo8;tab-stops:list .5in'><b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Trusts</span></b><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>: Trusts are used to define
     relationships between domains in the same forest or in different forests.
     Trusts provide a mechanism for sharing resources and enabling users in one
     domain to access resources in another domain.<o:p></o:p></span></li>
</ul>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><span lang=EN-IN style='mso-ansi-language:EN-IN'>Each of
these components plays an important role in the management and operation of
Active Directory. By understanding these components, administrators can
effectively manage and secure their directory services.<o:p></o:p></span></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><b>Group Policy: </b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><span lang=EN-IN style='mso-ansi-language:EN-IN'>Group
Policy is a feature of Active Directory that enables administrators to define
and enforce policies and settings for users and computers in a Windows domain.
Group Policy settings are stored in the Group Policy Objects (GPOs), which are
applied to Active Directory objects, such as users, computers, and groups.<o:p></o:p></span></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><span lang=EN-IN style='mso-ansi-language:EN-IN'>Group
Policy enables administrators to manage a wide range of settings, including
security settings, software installation, desktop configuration, and network
connections. Some of the key features of Group Policy in Active Directory
include:<o:p></o:p></span></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ol style='margin-top:0in' start=1 type=1>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Centralized management: Group
     Policy enables administrators to manage policy settings for the entire
     domain or for specific groups of users or computers from a single
     location.<o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Policy inheritance: Group
     Policy settings can be inherited from parent objects in the Active
     Directory hierarchy, providing a mechanism for defining default settings
     and exceptions to those settings.<o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Filtering and targeting: Group
     Policy provides a mechanism for filtering and targeting policy settings
     based on user and computer attributes, such as group membership,
     organizational unit, and IP address.<o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Security settings: Group Policy
     enables administrators to define and enforce security settings, such as
     password policies, account lockout policies, and user rights.<o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Software installation: Group
     Policy can be used to deploy and manage software installations across the
     domain.<o:p></o:p></span></li>
 <li class=MsoNormal style='mso-list:l6 level1 lfo9;tab-stops:list .5in'><span
     lang=EN-IN style='mso-ansi-language:EN-IN'>Desktop configuration: Group
     Policy provides a mechanism for configuring desktop settings, such as the
     wallpaper, screensaver, and Start menu.<o:p></o:p></span></li>
</ol>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><span lang=EN-IN style='mso-ansi-language:EN-IN'>Group
Policy is a powerful tool for managing and securing <span class=GramE>Windows
domains, and</span> is widely used in enterprise environments. By defining and
enforcing consistent policy settings across the domain, administrators can
ensure that their network is secure, compliant, and running efficiently.<o:p></o:p></span></p>

<p class=MsoNormal><b>Transitive trust: </b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>A transitive trust is when a trust can be extended outside
of the two domains in which it was created. A domain connected via a transitive
trust can thus access any other domain when there is a path of transitive
trusts between that domain and the target domain.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>Transitive trust is a two-way relationship automatically
created between parent and child domains in a Microsoft <span class=GramE>Active
Directory forest</span>. When a new domain is created, it shares resources with
its parent domain by default, enabling an authenticated user to access
resources in both the child and parent. <o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>The transitive property of equality states that if a = b and
b = c, then a = c. &nbsp;In an Active Directory transitive trust relationship,
if domain A trusts domain B, and domain B trusts domain C, then domain A trusts
domain C.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>Examples: infra-forest trust, tree - root trust,
parent-child trust<o:p></o:p></p>

<p class=MsoNormal><b>Shortcut Trust:</b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>Shortcut Trust&nbsp;is an <span class=GramE>explicitly
created,</span> transitive&nbsp;trust&nbsp;between two domains in a forest to
improve user logon times.&nbsp;Shortcut Trust&nbsp;will make <span class=GramE>a&nbsp;trust</span>&nbsp;path
shorter between two domains in the same forest. The&nbsp;Shortcut
Trust&nbsp;can be one-way or two-way.<o:p></o:p></p>

<p class=MsoNormal><b>Parent-Child Trust</b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>A transitive, two-way parent-child trust relationship
automatically <span class=GramE>created</span> and establishes a relationship
between a parent domain and a child domain whenever a new child domain is
created using the AD DS installation process <span class=SpellE>process</span>
within a domain tree. They can only exist between two domains in the same tree
with the same contiguous namespace. The parent domain is always trusted by the
child domain. You cannot manually create a Parent-Child trust.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><b>Tree-Root Trust:</b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>A transitive, two-way tree-root trust relationship
automatically created and establishes a relationship between the forest root
domain and a new tree, when you run the AD DS installation process to add a new
tree to the forest. A tree-root trust can only be established between the roots
of two trees in the same forest and are always transitive. You cannot manually
create a tree-root trust.<o:p></o:p></p>

<p class=MsoNormal><b>Shortcut Trust:</b><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>Shortcut trusts are manually created, one-way, transitive
trusts. They can only exist within a forest. They are created to optimize the
authentication <span class=GramE>process</span> shortening the trust path. The
trust path is the series of domain trust relationships that the authentication
process must traverse between two domains in a forest that are not directly
trusted by each other. Shortcut trusts shorten the trust path.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>A&nbsp;shortcut trust&nbsp;is an
AD&nbsp;trust&nbsp;relationship that administrators can explicitly define in
addition to the&nbsp;trust&nbsp;relationships that AD automatically creates
between the domains in an AD forest as part of the AD installation process (<span
class=SpellE>dcpromo</span>).<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<ul style='margin-top:0in' type=disc>
 <li class=MsoNormal style='mso-list:l7 level1 lfo10;tab-stops:list .5in'>Used
     to reduce access times in complex trust <span class=GramE>scenario</span>.<o:p></o:p></li>
 <li class=MsoNormal style='mso-list:l7 level1 lfo10;tab-stops:list .5in'>Can
     be one-way or two-way transitive.<o:p></o:p></li>
</ul>

<p class=MsoNormal><b>External trust:</b><span style='mso-spacerun:yes'>
</span><o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><span class=GramE>An&nbsp;external</span> trust&nbsp;is
a&nbsp;trust&nbsp;between domains in different forests.&nbsp; An external trust
is a one-way, non-transitive trust that is manually created to establish a
trust relationship between AD DS domains that are in different forests, or
between an AD DS domain<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>External trusts allow you to provide users access to
resources in a domain outside of the forest that is not already trusted by a
Forest trust.<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal>&nbsp;<o:p></o:p></p>

<p class=MsoNormal><b>Forest Trust:</b> Forest trusts are manually created,
one-way transitive, or two-way transitive trusts that allow you to provide
access to resources between multiple forests. Forest trusts <span class=GramE>uses</span>
both Kerberos v5 and NTLM authentication across forests where users can use
their Universal Principal Name (UPN) or their Pre-Windows 2000 method (domain
Name\username). Kerberos v5 is attempted first, and if that fails, it will then
try NTLM.<o:p></o:p></p>
